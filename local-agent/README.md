# LocalHelper - 本地文件助手

安全的本地文件管理系统，让 Web 应用能够使用系统默认应用打开本地文件。

## 功能特性

- 🔒 **文件本地存储** - 文件不上传云端，只存储路径引用
- 📂 **系统默认应用打开** - 使用本地的 Excel、Word、CAD 等应用打开文件
- 🌐 **B/S 架构兼容** - 保留 Web 应用的更新优势
- ⚡ **静默唤醒** - 通过自定义协议自动启动

## 系统要求

- Windows 10/11 (其他平台开发中)
- Chrome、Edge 或 Firefox 浏览器

## 安装

### 方式一：下载安装包（推荐）

1. 下载 [LocalHelper_Setup_v1.0.0.exe](../../dist/LocalHelper_Setup_v1.0.0.exe)
2. 运行安装程序
3. 按照提示完成安装

### 方式二：从源码编译

#### 前置条件

- Go SDK 1.21+
- Windows 10/11

#### 编译步骤

```bash
cd local-agent
.\build.bat
```

编译完成后，会在 `local-agent` 目录下生成 `LocalHelper.exe`

## 使用方法

### 首次使用

1. 安装 LocalHelper 后，首次打开支持本地文件的应用时会检测到助手未运行
2. 点击提示框中的链接下载安装包
3. 安装完成后刷新页面即可使用

### 日常使用

1. 打开支持本地文件的应用
2. LocalHelper 会在后台自动运行
3. 点击"添加本地文件"选择文件
4. 点击文件名即可用系统默认应用打开

## 技术架构

```
┌─────────────────┐          ┌─────────────────┐
│   云端 Web       │ WebSocket │  LocalHelper    │
│  (Supabase)     │◄─────────►│  (本地托盘)      │
│                 │ws://127   │                 │
│  - 存储 fileID  │ .0.1:     │  - SQLite 存路径 │
│  - 存储路径引用  │ 18080     │  - Shell 打开   │
└─────────────────┘           │  - 自定义协议   │
                              └─────────────────┘
```

## API 接口

LocalHelper 提供以下 WebSocket 命令：

| 命令 | 说明 | 参数 |
|------|------|------|
| `ping` | 心跳检测 | - |
| `pick_file` | 打开文件选择对话框 | - |
| `open_file` | 用系统默认应用打开文件 | `{id: "file_id"}` |
| `get_files` | 获取已管理文件列表 | - |
| `delete_file` | 删除文件记录 | `{id: "file_id"}` |
| `check_status` | 检查助手状态 | - |

## WebSocket 地址

```
ws://127.0.0.1:18080
```

## 安全说明

- 所有文件存储在用户本地
- 仅传输文件路径（不含文件内容）
- 使用自定义协议 `localhelper://` 唤醒应用
- WebSocket 仅监听本地连接

## 常见问题

### Q: 为什么需要安装本地助手？

A: 由于浏览器安全限制，网页无法直接调用系统应用打开本地文件。本地助手作为桥梁，实现安全、本地化的文件管理。

### Q: 本地助手安全吗？

A: LocalHelper 只存储文件路径，不访问或传输文件内容。所有操作都在本地完成，不会有任何数据上传。

### Q: 如何卸载？

A: 通过 Windows 设置 > 应用 > LocalHelper > 卸载即可。
